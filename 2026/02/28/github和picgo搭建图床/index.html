github和PicGo搭建图床

发表于2026-02-28|更新于2026-02-28
github+picgo搭建图床

起因

之前在网上找了个免费的公共图床使用，一开始还不错，除了加载的有点慢，但是过了几个月之后，可能是因为图床很久没使用，图片有些都消失了，于是重新找方法。

方法

由于我的博客用的是github+hexo，自然而然，我就想继续使用github继续当做存储空间，毕竟能存代码，也能就够存图片。 然后上网发现有picgo+github的图床，操作也十分简单，只需要开一个新的github仓库，然后生成一个允许细粒度访问这个仓库的token把token给到pikgo，在picgo里面配置好文件，就实现了一个图床。 而且这个图床速度快，容量限制也小，只有100 MB的图片限制，之前的公共图床图片限制在10 MB。不过也有缺点，就是仓库的总容量超过1 GB，会受到管理员的审查。不过到时候把仓库再换个新的就好了。

具体流程

  1. 下载安装picgo软件，同时github上新开一个仓库（这个仓库要公开）
  2. 获取可以访问这个仓库的token：
    • 登录 GitHub，点击右上角头像 → Settings
    • 在左侧导航栏选择 Developer settings
    • 选择 Personal access tokens
    • 点击 Generate new token（建议使用细粒度token，Fine-grained tokens
    • 设置 Token 名称，如果是Fine-grained tokens，还可以设置哪个仓库哪些权限可以使用（Note：建议合理设置有效期）
    • 创建后复制 Token 值并妥善保存（过期后需重新生成）
  3. 配置picgo：
    • 图床设置里选择Github图床
    • 仓库名用刚才创建的仓库“用户名/仓库名”
    • 分支名一般是master，我的是master，如果你是main就用main
    • Token就是刚才创建的一长串
    • 存储路径，意思是图片上传到的位置，如果没有该文件夹会自动创立
    • 自定义域名，推荐使用CDN加速的，格式“https://cdn.jsdelivr.net/gh/[GitHub用户名]/[仓库名]”。如果你是main分支后面加个“@main”
这样就ok了，可以传图片验证一下
