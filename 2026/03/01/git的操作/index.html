git操作

建议去这个网站学习

本地基础使用

  • 本地操作，首先 git init 进行初始化。然后要连接远程库git remote add origin git@github.com:用户名/仓库名.git后面才能用。也可以用 git clone。
  • 第一个疑问肯定是怎么上传我们的修改——git commit。它会把我们对文件的修改上传到本地的 git 仓库里，使用-m 参数可以添加此次的修改信息，使用--amend 参数可以将这次的修改合并到上一个提交中。注意每次commit 前都要将要提交的东西添加到暂存区，即git add 文件。
  • 那怎么撤销修改呢——git reset 和 git revert。reset 是直接删除指定分支（一个分支可以理解为一次修改上传），而 revert是创建一个新的分支，在该分支下，分支内容里就是没有之前修改的。
  • 那当我们和别人共同开发时，想互不干扰地上传时怎么办——git branch 分支名。它会在当前分支创造一个分支（可以理解为副本），我们可以自由上传修改这个分支而不影响其他分支。
  • 那如果想在其他分支那里修改怎么办——git checkout 分支名。它会切换到指定的分支去，使用-b 参数可以在分支不存在时创建该分支并切换。
  • 那如果想要把其他分支的修改和当前合并怎么办——git merge 分支名。它会在当前分支下把其他分支的修改合并，有冲突的地方要手动确认。
  • 那如果想要调整分支顺序或者将别人分支挪到自己分支下怎么办？这里先讲 HEAD和相对引用：HEAD 类似一个 C 指针，它指着 git 当前所在的分支；相对引用有^ 和 ~，比如 HEAD^ 和HEAD~ 都是指代上一个分支，HEAD~3 指代向上3 个分支，HEAD^2 指代第二个父分支（分支可能有多个父分支）。 然后将其他分支移到自己分支下——git cherry-pick 分支名1 分支名2 ...和 git rebase 开始分支名 结束分支名。我个人更喜欢cherry-pick，只需要把分支名一列就可以了，分支名顺序也是连接的顺序；rebase适合在想要迁移的分支名很长时使用，rebase 是创造一个更线性的链，不过也可以通过-i 参数互动式操作。
  • 还有一些常用的：git log 显示日志，git tag给分支一个固定的标签。

远程操作

  • 首先理解 git 如何管理本地和远程分支：
    1. 本地分支就是你电脑上存储的，远程分支就是远程仓库里存储的，git会额外在本地仓库创建一个远程跟踪分支去镜像远程分支，可以理解为远程分支的缓存，例如origin/main 就是 main 的跟踪分支。
    2. 在你下载、上传分支时，远程分支就是通过这个跟踪分支，知道要与哪一个分支去交互，每次上传下载也会更新这个跟踪分支。
    3. 也可以人为更改这种跟踪关系，使用git checkout 分支名 跟踪分支名 或git branch -u 跟踪分支名 分支名。
  • 其次就是上传、下载远程分支——git push [远程主机名] [本地分支名]:[远程分支名]、git pull [远程主机名] [远程分支名]:[本地分支名]、git fetch [远程主机名] [远程分支名]:[本地分支名]。
    1. git push就是上传本地分支到远程，如果本地分支和远程分支名一样，则可以省略冒号和远程分支名；如果本地分支名省略，则是删除远程分支；一般来说origin 也可以省略，直接写成 git push，将当前分支的提交推送到其跟踪的远程分支。这些规则除了删除，同样适用于fetch、pull。
    2. git fetch是下载远程分支到本地，但不会更新当前分支，还要人为合并或者 rebase 一下。
    3. git pull 是 git fetch + git merge 的结合，它下载远程分支后会同时合并到当前分支，很省事。
